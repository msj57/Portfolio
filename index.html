01 NLP · MLOps

Argos · Bene Gesserit Clasificación multilabel de crónicas históricas Multilabel classification of historical chronicles

RolRole — diseño y desarrollo end-to-end: arquitectura, entrenamiento, API de inferencia y frontend. end-to-end design and development: architecture, training, inference API and frontend.

Plataforma MLOps completa que clasifica crónicas históricas en cinco categorías mediante fine-tuning de un BERT en español exportado a ONNX, con tracking de experimentos en MLflow, etiquetado asistido con revisión humana y un pipeline de audio que transcribe con Whisper y reutiliza el mismo clasificador. Complete MLOps platform classifying historical chronicles into five categories through fine-tuning of a Spanish BERT exported to ONNX, with MLflow experiment tracking, human-in-the-loop assisted labeling and an audio pipeline that transcribes with Whisper and reuses the same classifier.

  • Siete servicios orquestados con Docker Compose: Next.js 15, NestJS 11 (REST + SSE + BullMQ), microservicio Python de ML, PostgreSQL, Redis, MLflow y Adminer. Seven services orchestrated with Docker Compose: Next.js 15, NestJS 11 (REST + SSE + BullMQ), Python ML microservice, PostgreSQL, Redis, MLflow and Adminer.
  • Un único modelo ONNX sirve texto y audio: extracción de PDF, chunking configurable e inferencia con probabilidades y confianza por fragmento, exportable a CSV. A single ONNX model serves text and audio: PDF extraction, configurable chunking and inference with per-chunk probabilities and confidence, exportable to CSV.
  • Ciclo de mejora continua del dataset: panel de análisis exploratorio (balance de clases, co-ocurrencia, TF-IDF discriminante) y revisión fragmento a fragmento para reentrenar con datos validados. Continuous dataset-improvement loop: exploratory analysis panel (class balance, co-occurrence, discriminant TF-IDF) and chunk-by-chunk review to retrain on validated data.

PyTorch/BERT/ONNX/MLflow/Whisper/FastAPI/NestJS/Next.js/PostgreSQL/Docker

Mobile health app with an agent that analyzes food photos with multimodal vision, computes macros, generates personalized meal plans and routines, and chats via streaming with long-term user memory.

RAG memory with pgvector: 768-dimension embeddings; the chat builds three-layer context (profile, recent meals, semantic retrieval) and emits embedded cards via SSE.

Complete product: TDEE calculation with dynamic targets, workout sessions with per-set progress, streaks and daily adherence. Strict TypeScript on the client.

03 Recomendación · Big DataRecommenders · Big Data

SoundWave Recomendador musical por emoción Emotion-based music recommender

RolRole — ingeniería de datos, motor de recomendación, API y frontend. data engineering, recommendation engine, API and frontend.

Sistema que clasifica más de 1,2 millones de canciones de Spotify por emoción a partir de sus audio features y recomienda por similitud coseno, con SPA en Angular y login OAuth de Spotify. System that classifies over 1.2 million Spotify songs by emotion from their audio features and recommends via cosine similarity, with an Angular SPA and Spotify OAuth login.

  • Ingesta de 1.204.022 canciones — limpieza, deduplicación y clasificación emocional sobre energy, valence, danceability, tempo y acousticness — en ~30–60 segundos. Ingestion of 1,204,022 songs — cleaning, deduplication and emotion classification over energy, valence, danceability, tempo and acousticness — in ~30–60 seconds.
  • MongoDB dockerizado e indexado por emoción para servir el motor de similitud coseno con latencia interactiva desde FastAPI. Dockerized MongoDB indexed by emotion serving the cosine-similarity engine at interactive latency from FastAPI.
  • OAuth 2.0 con PKCE contra la API de Spotify sin exponer secretos en el cliente, además de autenticación local. Documentación técnica con MkDocs. OAuth 2.0 with PKCE against the Spotify API without exposing client secrets, plus local authentication. Technical documentation with MkDocs.

Python/Pandas/MongoDB/FastAPI/Angular 21/OAuth PKCE/Docker/MkDocs
